# Clickeen AI

STATUS: REFERENCE - MUST MATCH RUNTIME

This folder documents Clickeen's AI plane and agent homes.

It is not a dumping ground for AI-written guides, rollout history, generated
runtime files, or product service docs.

## Product Law

Clickeen is an agent-operated product. The product codebase stays lean and
schema-structured so agents can operate structured artifacts directly.

AI ownership is simple:

```text
San Francisco executes governed model calls.
Ombra names the Clickeen product-AI layer and model strategy.
Built agent homes operate structured product artifacts.
Planned agents are future scope until code/runtime exists.
Documentation explains the system; documentation is not runtime input.
```

## What Does Not Belong Here

- AI coding guides for widgets or Prague page JSON.
- Generated runtime JSON.
- Product service docs for Roma, Bob, Tokyo-worker, Berlin, Prague, Dieter,
  Michael, DevStudio, or Tokyo.
- Rollout history that is not current runtime truth.
- Planned agents mixed beside built agents.

Generated model-maintenance outputs are not current documentation authority and
must not be read by San Francisco from `documentation/`. Future model catalog
monitoring and update work belongs to the planned DevOps Agent.
