# Dieter - Design System And Static Media

STATUS: CURRENT SYSTEM OPERATOR SPEC

Dieter is Clickeen's shared design system. It owns tokens, component CSS, component specs, component snippets, icons, and optional component hydrators. Product surfaces consume Dieter artifacts; account data never lives in Dieter.

## Runtime Authority

| Concern | Current authority |
| --- | --- |
| Source | `dieter/` |
| Build script | `scripts/build-dieter.js` |
| Package | `@ck/dieter` |
| Repo deploy output | `tokyo/product/dieter/**` |
| R2 deploy root | `dieter/**` |
| Public friendly path | `/dieter/**` |
| DevStudio preview | `admin/` generated pages |

`dieter/components/registry.json` is not runtime-authoritative for the shipped artifact manifest. The artifact manifest is generated by `scripts/build-dieter.js` at:

```text
tokyo/product/dieter/manifest.json
```

## Source Layout

| Path | Purpose |
| --- | --- |
| `dieter/tokens/` | Canonical token source. |
| `dieter/components/{component}/` | Component CSS, snippet, spec, and optional JS. |
| `dieter/icons/svg/` | Source SVG icons. |
| `dieter/icons/svg_new/` | Optional icon override input; build copies it over `dieter/icons/svg/` before normalization when present. |
| `dieter/icons/icons.json` | Generated/source icon registry used by build output. |
| `tokyo/product/dieter/` | Generated deploy artifacts. |

Component folders normally contain:

```text
{component}.css
{component}.html
{component}.spec.json
{component}.js   # optional static hydrator copied directly
{component}.ts   # optional hydrator bundled when basename matches folder
```

## Artifact Contract

The Dieter build emits:

- token CSS and matching shadow-token CSS
- component CSS
- component snippets/specs needed by consumers
- optional JS hydrators
- icons and icon registry
- `manifest.json`

Token outputs include `tokens.css`, `tokens.shadow.css`, and per-token CSS
files with matching `*.shadow.css` variants where the build creates them.

The manifest tells consumers which components have JS and which dependency bundles are required. Component CSS class names do not add JS bundles by themselves.

## Consumer Boundaries

Bob, Roma, Prague, DevStudio, and generated public widget packages may consume Dieter artifacts. They do not own Dieter source truth.

Bob/Roma own account context, auth, persistence, asset routes, and policy. Dieter owns primitive UI behavior. Dieter controls may request host actions through explicit host transports, but Dieter must not create account routes or write account folders.

Account-uploaded assets and fonts live under:

```text
accounts/{accountPublicId}/...
```

They do not live under root `dieter/`.

## Build And Verification

From repo root:

```bash
pnpm build:dieter
pnpm --filter @ck/dieter typecheck
pnpm dieter:governance:check
```

When Dieter changes affect DevStudio previews, also run:

```bash
pnpm --filter @clickeen/devstudio generate
pnpm --filter @clickeen/devstudio build
```

Artifact reconciliation:

1. Source changed under `dieter/**`.
2. Generated output changed under `tokyo/product/dieter/**`.
3. `tokyo/product/dieter/manifest.json` includes the expected component, JS, and dependency entries.
4. DevStudio rendered foundations/components show the expected result.

For deployed R2 artifact evidence:

```bash
pnpm cf:preflight
```

Then use the R2 commands documented in `documentation/engineering/CloudflareOperations.md`.

## Build Hard Stops

The Dieter build fails or must be stopped when:

- `dieter/tokens/tokens.css` is missing
- the icon SVG directory is missing
- SVG normalization or verification fails
- a component has unsupported artifact shape
- generated `manifest.json` is missing or inconsistent
- generated output would write outside `tokyo/product/dieter/**`

## Operator Rules

- Update source first, then rebuild generated Tokyo artifacts.
- Do not hand-edit generated artifacts without the source change that produces them.
- Do not add external vertical margins to reusable controls; host layout owns outside spacing.
- Do not use `@import` for fonts in generated widget packages.
- Public widget artifacts inline required SVG markup during package generation; they do not fetch icon bundles at runtime.
- Do not claim accessibility, privacy, retention, or compliance guarantees here unless backed by current source and tests.
