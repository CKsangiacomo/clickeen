# Strategy Documentation

STATUS: INFORMATIVE - STRATEGY ROUTING

This folder contains directional operator knowledge: why Clickeen exists, what
moats matter, how the market is framed, and what product judgment must preserve.

It is not a current-system operator manual.

## What Belongs Here

- durable company thesis;
- moat explanations;
- market positioning;
- global/content strategy;
- operator rules that prevent agents from rebuilding legacy SaaS patterns.

## What Does Not Belong Here

- runtime routes;
- storage paths;
- worker contracts;
- database schema;
- billing/tier enforcement;
- feature status;
- execution slices;
- acceptance criteria;
- migration notes;
- old architecture claims pretending to be current;
- investor decks, pitch drafts, fundraising asks, and date-stamped proof claims.

Those belong in:

- `documentation/architecture/` for current platform boundaries;
- `documentation/services/` for current service behavior;
- `documentation/capabilities/` for current cross-system capability contracts;
- `documentation/ai/` for the AI plane and built agents;
- `documentation/widgets/` for widget operator manuals;
- `documentation/engineering/` for runbooks;
- `Execution_Pipeline_Docs/` for planning, execution records, history, and
  non-current collateral.
