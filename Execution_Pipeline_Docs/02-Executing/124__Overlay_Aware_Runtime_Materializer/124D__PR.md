## Verdict

**Green-light (plan).** 124D materializes active non-base locale artifacts from
exact source plus one Babel overlay, stores them as derived package bytes, and
correctly **does not expose public serving** (124E owns that). It honors the
no-reinterpretation tenet (explicit forbidden-additions list: no alternate
documents, no request-time composition, no fallback/precedence, no A/B/
personalization, no public serving, no compatibility readers, no repair), keeps
authorities intact (Roma commands, Translation Agent writes overlays only, Tokyo
stores, materializer resolves), and gets the partial-failure contract right
(Slice 5: completed/skipped/failed coordinates, no full-success lie, no rollback —
V6). One consistency carry-forward + a few closeout conditions.

## Slice-by-slice + closeout conditions

- **Precondition Gate + Slice 1 (closed-contract check):** sound. Blocks if
  124A/124B/124C values aren't locked; "the agent must not infer the missing
  contract."
- **Slice 2 (overlay-key compatibility):** sound. The active branch per the 124A
  addendum §5.4 is **concrete paths accepted, no Translation Agent key change**
  (step 1). **Carry-forward (same as 124B):** the "repeated fields deferred"
  wording (steps 3, Slice 7) must be scoped to **reorder-safe token cascade**
  specifically — 124A §5.4 allows concrete-path repeated fields. Align the
  wording so an implementer doesn't reject all repeated locale materialization.
  Slice 2 step 4 (old-shape overlays fail explicitly, regenerate, no dual readers)
  is correct V5/V7 discipline.
- **Slice 3 (Roma locale materialization command):** sound. Reuses 124C's
  media-resolution step; active-locale + non-base checks; one overlay. **Condition:**
  the Roma→materializer input must carry the same byte-stable state-serialization
  path as 124C (locale artifacts go through the same `materializeConfigMedia`→
  `JSON.stringify` path; byte-stability applies to locale artifacts too, not just
  base).
- **Slice 4 (Tokyo locale package storage):** sound. Stores derived bytes at the
  124A coordinate; no public route in 124D; overlay file untouched. Writes
  authorized by Roma capsule, not Translation Agent grant (correct — the package
  write is a Roma command, not an agent write).
- **Slice 5 (translation generation integration):** strong V6. After overlay
  generation succeeds, materialize locale artifacts in deterministic order;
  first-failure stops; completed/skipped/failed coordinates; no rollback of prior
  successes; no full-success claim unless every locale has both overlay + package.
  This correctly extends existing Babel failure law.
- **Slice 6 (active-locale settings):** sound. Removed locale → delete package
  artifact (follow-up failure reporting if delete fails); added locale → generate.
  Account settings remain the user's truth; artifacts are follow-up. Consistent
  with existing overlay-deletion behavior.
- **Slice 7 (failure contract):** comprehensive, V1/V3 clean. No base-as-locale,
  no missing→base substitution, no drop-unexpected.
- **Slices 8–10 (tests, verification, closeout):** sound. Closeout fence: no
  public locale URL exposed, no alternate document, Translation Agent writes
  overlays only.
