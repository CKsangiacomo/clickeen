## Verdict

**Green-light (plan).** 124F wires the **executable-now** cascade subset (source
save, overlay write, active-locale add/remove, byte-affecting policy) as
**command-owned, coordinate-exact** operations — no dependency graph, event bus,
scheduler, watcher, or broad fan-out (all correctly forbidden and deferred to
124G). It honors the no-reinterpretation tenet and has strong V6 discipline
(first-failure-stops, completed/skipped/failed coordinates, no full-success lie,
old overlays not treated as current for changed source). One real watch-item
(sync cascade cost) + carry-forward dependencies. No blockers.

## Slice-by-slice + closeout conditions

- **Cascade Law + forbidden additions:** sound. "source command changes truth →
  same command resolves affected coordinates." No background jobs/watchers/status
  files — the cascade lives inside the command that mutated truth.
- **Slice 1 (inventory):** sound. Each command must name affected coordinates or
  stop — no inferred discovery.
- **Slice 2 (superseded source-save follow-up) — the crux + watch-item:**
  - **Closeout condition 1 (sync cascade cost — corrected in round 1).** The real
    *live* synchronous fan-out is **Slice 5**, not Slice 2: `reconcileAccountLocaleOverlays`
    in `roma/app/api/account/locales/route.ts` loops over **every account instance ×
    added locales**, calling Translation Agent synchronously in one edge request —
    that is the actual latency/timeout surface today. Slice 2 superseded source-save follow-up is
    **greenfield** (current save returns a bare `{ok:true}`, no locale work), so 124F
    is the first time generation runs *inside* the save request. **Bound it:** the
    cost surface is **instance-count × locale-count** (not per-instance); document the
    latency ceiling (worst case = tier2-4 account at the 28-locale cap with a large
    field count) against the Cloudflare request budget, and name async (queue/Worker)
    as the escape valve when it's exceeded (the 124G/later path). Don't let a
    high-instance/high-locale account silently time out and report partial-success.

## Vectors and blast radius

| Vector | Failure mode | Blast radius |
| --- | --- | --- |
| Sync cascade timeout (high-locale account) | save partially completes; partial-success risk | per-save, account-scoped |
| Locale-coordinate unresolved | Slice 5 delete path undefined | blocks 124F Slice 5 |
| Old overlay applied to changed source | stale locale served as current (V1) | per-locale, silent until served |
| Removed-locale artifact not deleted | stale locale stays reachable | per-locale, bounded by deletion |
| Partial-success claimed | user told all locales current when some failed (V6) | per-save |
| Duplicate copies locale authority | clone-model leak (overlays/packages copied) | per-duplicate |

Blast radius: **the cascade path** — source save now affects base + all active
locale artifacts for an instance. High at the per-save level; bounded by locale
cap; mitigated by command-ownership + V6 response contract.
