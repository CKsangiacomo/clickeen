## Verdict

**Green-light (plan).** 124C is the first runtime-touching slice, correctly framed
as a **boring, byte-identical reroute**: Roma's base package generation calls
`@clickeen/ck-runtime-materializer` instead of its inline builder, producing the
same `index.html/styles.css/runtime.js` and the same Tokyo stored-bytes contract.
It honors the no-reinterpretation tenet (explicit forbidden-additions list:
no request-time composition, no URL/storage migration, no compatibility readers,
no repair/probes) and keeps Tokyo 100% unchanged (Slice 5). Five binding closeout
conditions + one execution prerequisite, all in-scope.

**Blast-radius shift:** 124A/124B were zero-runtime-blast (docs / unwired
package). 124C changes Roma's live create/save/duplicate path for every account.
A byte-drift bug is user-visible on the next save, and there is **no canary** for
a build-path change — parity proof is the only net.

## Execution prerequisite

**124B must be code-delivered, not just plan-green.** 124C Slice 2 step 1
self-gates on `packages/ck-runtime-materializer` existing in the workspace.
Plan-green ≠ code-delivered (the MAMA/124B-gate distinction). Roma can't be
rewired until the package is built.

## Slice-by-slice + binding closeout conditions

- **Slices 1–2 (inventory, wiring):** sound. `compileWidgetForInstancePackage`
  (Bob route) and `materializePublicPackageMedia` (Tokyo asset control) correctly
  stay in Roma; the Roma→materializer edge is the only new edge, no reverse edge.
- **Slice 3–4 (adapter + reroute) — the crux:**
  1. **Byte-exact parity targeting the state-serialization path.** Drift risk is
     the *state object's key order*, not the adapter wrapper: `materializeConfigMedia`
     (`ck-contracts`) reconstructs every node via `Object.entries`+fresh literals
     → insertion-ordered `state` → `JSON.stringify({instanceId, baseLocale, locales})`
     into `runtime.js` (`account-instance-public-package.ts:295-302`) → Tokyo SHA-256
     (`package-files.ts:93-104`). Byte-safe path = pass the already-stringified
     payload through untouched, OR prove canonical form equals today's
     `JSON.stringify`. Semantic parity is not enough.
  2. **Dual-build across all widget types** in cloud-dev, not a synthetic fixture —
     `stampPackageRoot`/`resolveProductPath` are widget-shape-sensitive.
- **Slice 5 (Tokyo unchanged):** correct cohesion call — a storage-contract change
  is CDN-wide blast radius; a boring reroute must not mutate it.
- **Slice 6 (error mapping):** sound V1/V3/V6. **Condition 3:** the materializer's
  error types → `InstancePackageFailure` mapping must be **total** (every error →
  a known reason key; no internal leak, V7).
- **Slice 7 (tests):** golden-fixture rule is right. **Condition 4 (harness):**
  the dual-build/golden parity needs a runner — 124B adds `tsx` to the materializer
  package; **124C must decide the Roma-adapter test surface** (a Roma `tsx` runner,
  or defer Roma-side unit tests to the materializer-hosted parity + e2e). **Condition
  5 (breadth):** parity across all widget types, not one.
- **Slice 8 (runtime verification):** **Condition 6 — deploy gated on parity**
  (no canary for a build-path change; every save flips on deploy).
- **Slice 9 (closeout):** correct fence (no locale artifact/URL; Tokyo fingerprint
  unchanged; evidence not persisted as product-status).

## Round-1 gate refinements (folded in; non-blocking)

Triple-GREEN round 1. Two Principal-TPM refinements:
- **Roma-adapter test harness is a Slice-3 prerequisite, not just a closeout
  condition.** The adapter IS the reroute; an undecided harness means parity
  cannot be proven at the moment it matters. Resolve before Slice 3 begins.
- **Name the contract version that pins the state-serialization path.**
  `materializeConfigMedia` lives in `@clickeen/ck-contracts`, outside 124B's
  `materializerContractVersion`. The byte-exact path depends on a contract-package
  version the materializer doesn't control — pin which version of `ck-contracts`
  (or its `widgetEditableFieldsContractHash`) governs the `state`-serialization
  bytes, or the parity proof has an uncontrolled input.
