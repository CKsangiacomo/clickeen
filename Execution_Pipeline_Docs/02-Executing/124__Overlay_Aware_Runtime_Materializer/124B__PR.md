## Verdict

**Green-light**, with three binding closeout conditions (down from four — the
fingerprint hoist moved to 124C). 124B is the right shape for a pure-package
slice: it creates `packages/ck-runtime-materializer` as a pure resolver, changes
no runtime behavior, defers every 124A decision, and forbids itself from being a
Worker/service/agent/storage-writer/policy-engine. It honors the
no-reinterpretation tenet by *extracting* Roma's existing build logic rather than
inventing a renderer. Blast radius is low by design (the package is unwired until
124C) — the real risk is contract-correctness for 124C/124D.

## Slice-by-slice review (revised)

### Slice 1 — Scaffolding
Correct. **Closeout condition:** the "no forbidden imports" rule must be a
CI-enforced structural check (eslint/import-graph test), not manual review —
the package's entire value is its isolation. The allow/forbid line maps exactly
to shared-package-vs-service: ALLOW `ck-contracts`, `ck-policy`, `widget-shell`,
`l10n`; FORBID `roma`, `tokyo-worker`, `sanfrancisco`, `berlin`, `agents/*`,
`@clickeen/bob`, `next`, cloudflare runtime.

### Slice 2 — Roma builder inventory
Sound. **Closeout condition (name the media seam):** the pure/I/O boundary
*already exists inside Roma* — `buildSavedWidgetPublicPackage` is pure;
`materializePublicPackageMedia` does I/O via `fetchTokyoAssetControl`. Slice 2's
extraction map must name this seam explicitly so the package ports the pure half
and the caller (Roma/124C) owns media resolution. Don't rediscover it in Slice 5.

### Slice 3 — Input contract
Sound. **Closeout condition (compiled-widget type boundary):** the materializer
defines its OWN compiled-widget input type in the package — it must NOT import a
type from `@clickeen/bob` (a deployed service). Roma (124C) adapts Bob's compiled
output to the materializer's input type. This keeps the materializer Bob-free and
makes the input contract package-owned.

### Slice 4 — Output and evidence
- **124B emits its own deterministic evidence fingerprint** (SHA-256, per 124A
  method). It does NOT touch Tokyo and does NOT hoist anything.
- **Fingerprint-equality with Tokyo is a 124C-era decision**, not a 124B
  mandate. 124C's PRD (Slice 5) already authorizes the permissive path: 124B
  emits its fingerprint; if it differs from Tokyo's, it stays local to package
  tests until 124C reconciles (by hoisting, aligning, or having Tokyo trust the
  materializer). Mandating the hoist in 124B would (a) touch live Tokyo code,
  breaking 124B's no-runtime-touch authority gate, and (b) import a downstream
  decision into this slice — a no-reinterpretation tenet risk.
- **Two-hash rule (from 124A):** SHA-256 for package/artifact fingerprints;
  FNV-1a only for internal identity keys. Do not conflate.
- **Carry-forward to 124C (from Principal-TPM):** *when* 124C reconciles
  fingerprints via a hoist, it must move the **triple** — `PUBLIC_PACKAGE_FINGERPRINT_METADATA_KEY` constant + `buildInstancePublicPackageFingerprint` function + `SubmittedInstancePublicPackage` type — together into the shared package, or the SHA-256 payload drifts.

### Slice 5 — Base locale resolver
Sound. **Closeout condition (golden parity):** "preserve current behavior" cannot
be proven in 124B — Roma does not call the package yet (that is 124C). So 124B
must capture current Roma base output as **golden files** and assert byte-for-byte
reproduction in package tests. (Slice 8 currently omits this — it is the only way
124B can falsify "no behavior change" without being wired.) Live runtime parity is
124C's gate.

### Slice 6 — Locale overlay resolver
Correct. Defers repeated-field support to 124A's `scalar_only_initially`;
explicitly non-claims unsupported repeated fields (V3 guard, no silent omission).
Overlay-application semantics must match `previewL10n.js`'s `applyTranslatedValues`
exactly — owned by Slice 7. (Verified divergence today: `previewL10n.js`
silently skips on type mismatch; server path throws — a live V1/V6 vector the
parity work closes.)

### Slice 7 — Parity support
Sound. Inherits 124A's decision; conformance suite must be CI-gated (two
divergent resolver implementations today).

### Slice 8 — Tests
Good coverage. **Add:** golden-output parity test (Slice 5) and the
forbidden-imports structural test (Slice 1). **Resolve:** FNV-1a
`widgetEditableFieldsContractHash` (wire vs remove).

### Slice 9 — Closeout gate
Correct. 124B closes runtime-inert: no Roma path calls it (that is 124C), no
forbidden imports, fingerprint emitted as evidence only (not reconciled with
Tokyo — that is 124C).
