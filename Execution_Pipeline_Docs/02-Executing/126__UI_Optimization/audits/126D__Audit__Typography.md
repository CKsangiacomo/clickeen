## Current As-Built Read

Clickeen already has two real typography lanes:

- Dieter operational typography for Bob/Roma/Admin/DevStudio/Prague chrome and
  Dieter components.
- Public widget content typography through Bob-authored structured config and
  `CKTypography.applyTypography`.

The issue is not missing typography invention. The issue is drift between lanes
and false authorities:

- raw semantic HTML selectors acting as visual style authority;
- undefined or stale class names;
- widget font data using a global `/fonts/special/*` bucket instead of the
  account asset model;
- missing account font library data contract;
- root font route/proxy/config still present after the product moved custom
  fonts to account ownership;
- docs describing only part of the active system;
- runtime/editor data not perfectly aligned.

## Account Font Library Schema Audit

The account font library is top-level account metadata in
`accounts/{accountPublicId}/widget-defaults.json`. Its persisted records must
use exact enum values:

- `source`: `google` or `account-asset`.
- `category`: `sans`, `serif`, `display`, `script`, or `handwritten`.
- `familyClass`: `sans` or `serif`.
- `usage`: `body-safe` or `heading-only`.
- `styles`: `normal` or `italic`.
- `weights`: string weights `100`, `200`, `300`, `400`, `500`, `600`, `700`,
  `800`, or `900`.

Google records carry `spec`. Account-asset records carry `assetRef` and
`contentType`. Persisted records do not carry public CDN URLs or root
`/fonts/**` URLs. Malformed values fail validation; they are not normalized into
nearby values.
