## Current As-Built Read

The icon source set is healthy. The codebase already has the approved 157-icon
SF Symbols port, `currentColor` source SVGs, and matching generated Tokyo
output. 126C is not a missing-icons program.

The failures are operational and agent-facing:

- ordinary deploy build mutates source SVGs through `scripts/process-svgs.js`;
- root `scripts/build-dieter.js` still contains the inactive `icons/svg_new`
  override lane;
- an orphaned `dieter/scripts/build-icons.mjs` script plus docs describe a
  nonexistent `dieter/dist` registry;
- `.diet-icon` uses fake non-numeric size aliases and several aliases resolve
  to the same size;
- DevStudio/Admin skips missing icons silently in the global Admin
  `hydrateIcons` function in `admin/src/main.ts`;
- Bob emits inline SVG without decorative SVG attributes;
- Bob app chrome has named direct consumers in `Workspace.tsx`,
  `ToolDrawer.tsx`, and `TdMenu.tsx`;
- Prague consumes Dieter icons through `DieterIcon.astro`;
- `.github/workflows/cloud-dev-workers.yml` references
  `scripts/process-svgs.js`;
- public widget icon consumption is mixed and must be classified by file;
- docs do not tell agents one deterministic way to consume approved icons per
  consumer lane.
