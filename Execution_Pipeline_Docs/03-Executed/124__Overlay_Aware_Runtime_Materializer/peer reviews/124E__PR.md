## Verdict

**Green-light (plan).** 124E is the CDN/cost slice, correctly framed: public
traffic reads **stored evidenced bytes only** — Tokyo never reads overlays, calls
the materializer, asks Roma/Supabase, or composes on visitor requests. It honors
the no-reinterpretation tenet (explicit forbidden-additions list), is **honest
about immutable-vs-mutable cache** (immutable headers only for content-addressed
support files; short-TTL otherwise — no lying about immutability), and fails
closed on any missing/stale/mismatched locale evidence with **no base-as-locale**.
**Round-1 gate: 1 GREEN + 1 GREEN-conditional + 1 RED (Senior-PM) on four
blockers — see below. Serving-engine sound; RED is on product-ownership +
contract decisions the team must close before 124E ships to visitors.**

## Round-1 gate: blockers + findings (folded into Rev 2)

Round 1: Staff-engineer GREEN (2 corrections), Principal-TPM GREEN-with-conditions
(2 blockers reframed), **Senior-PM RED** (2 blockers). 124E is the first
visitor-facing localization surface, so the gate is stricter on product ownership.

**BLOCKERS (team decisions — PRD/PM edits, not mine):**
1. **Visitor-locale-miss UX is unpinned (PM).** Slice 3.10/7.8 make a missing/stale
   locale a bare `404`. The PM is right: the URL exists (instance published), only
   the locale is unavailable — a flat 404 tells visitors "doesn't exist" and
   crawlers "gone," both false. The right UX is an explicit "not available in this
   locale" page. 124E owns the *mechanism* (no wrong-language serve); the *product
   decision* (404 vs honest page) is a PM call to record in 124A before 124E ships.
   (Rev 1 overclaimed this as "RESOLVES" — it only resolves the mechanism.)
2. **Locale discovery is unowned — 124E serves URLs nothing can find (PM + TPM).**
   No slice in 124A–124H owns hreflang/sitemap/structured-data for the widget
   coordinate. 124E correctly refuses implicit discovery (Slice 2.6), but it puts
   localized bytes at a crawlable coordinate with no discovery path — reachable
   only if you already know the URL. Pin an owner before 124E ships.
3. **Locale-coordinate circular deadlock persists (TPM).** Rev 1 called this a
   "reinforcement"; it is a BLOCKER — 124E's own Precondition Gate items 1-2
   hard-stop until 124A names the locale URL + storage coordinate (the 124D
   unresolved deadlock). 124E cannot execute Slices 1-2 until 124A is amended.
4. **Two-tier CDN is conditional on an undecided contract (TPM).** Immutable
   support files require content-addressed URLs, but 124D never mentions
   content-addressing and 124A §9.2/§12 defers the immutable-support-file contract
   to 124E. So the "two-tier CDN" is aspirational; likely outcome is status-quo
   short-TTL (`clk-live-routes.ts:67-72`). 124E Slice 1 must OWN the
   content-addressed-support-file decision, not inherit it.

**Non-blocking corrections folded in:**
- (Staff-eng) Stale-reachable window is wider than Rev 1 stated: bounded by
  deletion AND the entry URL's 24h `stale-while-revalidate` tail — an edge that
  cached the entry may serve a removed locale up to 24h even after R2 deletion +

- (Staff-eng) The CDN-divergence honesty (immutable only when content-addressed)

- (TPM) `purgeClkLiveEntryCache` already accepts a discarded `locales?: string[]`
  param (`operations.ts:81,83`) — a stub awaiting 124E Slice 6; the executor is

**Verdict reassessment:** the serving-engine work is sound and disciplined (stored

## Slice-by-slice + closeout conditions

- **Slice 1 (URL/artifact lock):** sound. Self-gates on 124A naming URL + path +
  evidence; stops if ambiguous.
- **Slice 2 (route parser):** sound. Explicit locale URL only; rejects malformed
  + all implicit locale inference. **Condition:** preserve base + page parsing
  exactly (regression risk on the existing base path).
- **Slice 3 (publish + evidence gate):** strong V1/V5. Published pointer required;
  evidence must bind to account/instance/base-locale/requested-locale/package-fp;
  404 on any mismatch; step 9 stops if matching would require overlay reads/
  materializer/Roma/storage-walks. Correct: serving trusts stored evidence only.
- **Slice 4 (stored-byte response):** sound. Direct R2 bytes; `nosniff`;
  `GET`/`HEAD`; HTTPS redirect preserved.
- **Slice 5 (cache headers):** sound and honest. Immutable (`max-age=31536000,
  immutable`) only for content-addressed; short-TTL for mutable; all three header
  names consistent. Step 6 forbids visitor-time materialization to dodge cache.
- **Slice 6 (purge):** sound. Materializer out of purge; Roma-owned commands
  trigger via Tokyo executor OR short-TTL-only for locale entries (pick one);
  **step 8 handles the verified purge-config gap** (ZONE_ID/API_TOKEN unbound →
  explicit failure, no refresh claim); step 10 distinguishes "written but refresh
  failed" from full success. PreFlight-gated (`pnpm cf:api:preflight`).
- **Slice 7 (removed/stale):** sound + honest. **Note the stale-reachable window:**
  a removed-locale artifact stays publicly reachable until the account-owned
  deletion succeeds, because public serving must not query account active-locale
  state (step 3). This is the correct fail-closed-for-serving tradeoff (no
  visitor-time account lookups), but it's a known window — acceptable per the law,
  worth documenting as an explicit limitation.
- **Slices 8–10 (tests, verification, closeout):** sound. Dependency-guard test
  (no overlay/materializer/Roma/Supabase imports in `clk-live-routes.ts`) is the
  right purity fence.

## Vectors and blast radius

| Vector | Failure mode | Blast radius |
| --- | --- | --- |
| Base-path regression (Slice 2) | existing base serving breaks | all base public serving |
| Immutable header on mutable URL | stale bytes cached as immutable | per-support-file, long-lived |
| Base served for locale URL | wrong-language bytes (V1) | per-locale, silent |
| Locale-coordinate unresolved | 124E can't implement Slice 1-2 | blocks 124E execution |
| Stale-reachable window (removed locale) | deleted-but-not-yet-purged locale stays live | per-locale, bounded by deletion |
| Purge config unbound | refresh fails closed (503) | per-entry, fail-closed (correct) |

Blast radius: **public serving** — high (visitor-facing). Mitigated by stored-byte-
only + evidence gate + fail-closed + dependency guard.
