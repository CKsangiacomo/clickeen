# PRD 106 - Superseded Page Composer Planning Note

Status: Superseded
Owner: Product + Architecture
Date: 2026-06-04
Surviving authority: `../02-Executing/106__Umbrella__Composition_Vision.md`

This planning file used to carry the PRD 106 doctrine, but it grew stale after
the 2026-06-04 tenets correction. It is no longer active truth.

The active PRD 106 intent is simple:

- widgets are product-owned software;
- instances are account-owned widgets users create/edit in Roma/Bob and save to
  their account in Tokyo;
- pages are account-owned ordered stacks of instances stored in Tokyo;
- Bob edits one widget in browser memory and submits save;
- Roma owns account routing, tier/product permission, page composer UX, save
  acceptance, and publish/unpublish intent;
- Tokyo is responsible for R2 storage, nothing more.

PRD 106 does not introduce blocks, page sections, route maps, slugs, admin-only
branches, Bob page mode, or a Tokyo page composer.

When this note conflicts with the umbrella, the umbrella wins.
