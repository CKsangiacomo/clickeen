# PRD 107 - Withdrawn Page/Block Strategy Note

Status: Withdrawn
Owner: Product + Architecture
Date: 2026-06-04
Reason: Conflicts with PRD 106 page/instance tenets

This planning stub replaces the old PRD 107 page/block strategy. The withdrawn
version treated blocks as a future product layer for pages. That contradicts
the surviving PRD 106 model:

- pages are stacks of widget instances;
- page-shaped surfaces such as hero, split, steps, CTA, and card grids are
  widgets;
- there is no block product noun in the account authoring path.

Future SEO/GEO planning must start from the active PRD 106 authority:
`../02-Executing/106__Umbrella__Composition_Vision.md`.
